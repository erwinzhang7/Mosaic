# Mosaic asset catalog

## AppIcon

`AppIcon.appiconset/` holds the macOS app-icon slots. Every size is currently
filled with a **placeholder** generated by `scripts/gen-icon.swift` — a flat
blue rounded square with a 3×3 white dot grid. It's deliberately
unmistakable as a placeholder, original artwork, and not sourced from
anywhere external.

### Replacing the icon

To ship a real icon, replace the PNGs in `AppIcon.appiconset/`. All slots
use these filenames (rename your exports to match, or update
`Contents.json` accordingly):

```
icon-16.png    16×16   (1× for 16pt, also 1× for 32pt @ "32x32")
icon-32.png    32×32   (2× for 16pt, also 1× for 32pt)
icon-64.png    64×64   (2× for 32pt)
icon-128.png   128×128 (1× for 128pt)
icon-256.png   256×256 (2× for 128pt, 1× for 256pt)
icon-512.png   512×512 (2× for 256pt, 1× for 512pt)
icon-1024.png  1024×1024 (2× for 512pt)
```

No code changes needed — `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon`
in `project.yml` wires this catalog as the bundle icon.

### Menu-bar glyph (separate from the app icon)

The menu-bar status item uses an SF Symbol —
`NSImage(systemSymbolName: "square.grid.3x3.fill", ...)` — set in
`Sources/App/AppDelegate.swift` (`installStatusItem`). SF Symbols are
auto-template-rendered, so they adapt correctly to light/dark menu bars
without any extra work.

To swap the menu-bar glyph: either change the `systemSymbolName` argument
to a different SF Symbol, or replace it with a custom template image:

```swift
let img = NSImage(named: "MenuBarIcon")
img?.isTemplate = true   // critical — without this it won't adapt
button.image = img
```

If you go the custom-image route, add a `MenuBarIcon` image set to this
catalog with a single black-on-transparent PNG (the system handles the
inversion). Keep it small — 18×18pt at 1× / 36×36 at 2× is typical.
